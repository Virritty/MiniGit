#MiniGit

## Описание

Это реализация системы контроля версий на C, демонстрирующая основные принципы Git: неизменяемость коммитов, хэширование содержимого (улучшенный DJB2) и структурное разделение данных. Проект позволяет создавать коммиты, управлять файлами, работать с ветками и анализировать историю. Содержит модульные тесты и CLI-интерфейс.

## Основной функционал

- **Инициализация репозитория** — создание служебной директории `.minigit` с необходимой структурой
- **Добавление и изменение файлов** — создание новых версий с обновлённым содержимым
- **Удаление файлов** — исключение файлов из последующих версий
- **Фиксация изменений (коммиты)** — сохранение состояния с пользовательским сообщением
- **Просмотр истории** — вывод цепочки коммитов от текущего до начального
- **Получение содержимого файла** — извлечение любой версии файла из любого коммита
- **Проверка существования файла** — быстрая проверка наличия файла в коммите
- **Ветвление** — создание веток и переключение между коммитами (detached HEAD)
- **Структурное разделение** — автоматическое переиспользование неизменённых файлов между коммитами
- **Дедупликация данных** — одинаковое содержимое файлов хранится в единственном экземпляре
- **Подсчёт объектов** — анализ количества уникальных коммитов и блобов в репозитории
- **Оценка экономии памяти** — отображение процента переиспользованных объектов

## Сборка и запуск

Компиляция :
``` gcc minigit.c -o minigit.exe ```

Запуск:
``` ./minigit.exe ```

Компиляция и запуск тестов:
``` gcc test.c -o test.exe ```
``` ./test.exe ```
